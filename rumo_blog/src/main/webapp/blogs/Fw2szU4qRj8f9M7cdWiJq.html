现在程序主流都才用MVVM模式开发，产出RESTful 风格的 API 提供给客户端调用。但是除了他之外还有 GraphQL 这种方式。这里将介绍一下使用 Spring Boot 构建 GraphQL 程序 。

简介

GraphQL 的介绍这里不过多讲了，在我使用的经历看来，他是规范了前后端查询的一种协议，能够方便的动态变动每个请求接口中返回的数据。

比如查询文章列表这个API，在界面上有2个地方会使用它，其中一个需要使用文章标题及文章简介等信息，而另外一个则只需要使用到文章标题。在 RESTful 中如果只提供一个全信息的接口，那么直需要使用文章标题的地方不得不每次都查出来文章的介绍。这里问题就产生了，文章介绍不需要用到，但是每次 HTTP 请求都会返回这个结果，无疑的造成了网络层的流量浪费。那么你可以在请求的时候增加一个参数用来表示返回时需要什么字段（指定只需要返回 文章标题 字段），这样做的话就和 GraphQL 的作用极其相似。

最初发现 GraphQL 是一直在想找一个类似 JIRA 的 JQL 轮子，而无意间发现的。可惜 GraphQL 并未强大到那个地步。不过作为技术点学习一下思想还是蛮不错的。

官网：http://graphql.cn/

方便调试的工具，类似 Postman：https://github.com/graphql/graphiql
