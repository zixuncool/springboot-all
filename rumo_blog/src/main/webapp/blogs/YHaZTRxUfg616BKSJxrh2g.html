微软75亿喜提GitHub，并不一定是件坏事。

发表于 2018/07/19 14:05:39   |   分类于 :  前端   |   访客 : 269   评论 : 0

全世界最大的同性交友网站，哦不，全球最大的社交编程及代码托管网站 GitHub，刚刚过完十岁生日的他，昨天委身于微软了。

从上周末开始，微软收购GitHub的传闻就一直在不断发酵，一般这种收购传闻，在发酵一两天之后都会被证实为是真的。所以，果然不出所料。本周一，据彭博社报道，微软已经同意收购 GitHub 这个知名的代码托管网站。当晚，微软也在官网证实了这一消息。

小编到维基百科上看了一下关于GitHub的介绍，GitHub的owner已经从GitHub, Inc.变更为 Microsoft, Inc.

GitHub作为全球最大的社交编程及代码托管网站，这次的被收购，尤其是被微软收购到底是好事还是坏事呢？这又将对全球开发者产生怎样的影响呢？

被收购耻辱吗？

很多创业者，在创业初期都信誓旦旦的说，我的公司永远不要被收购，我要成为一个巨头。还有的人更狠，我不要上市，不要资本的钱。

而现实，往往很残酷，对于很多创业公司来说，最好的归宿可能就是被巨头收购了！

“被收购是一种耻辱”，一般是中国人特有的想法。因为从古至今，中国人最在意的就是“领土完整”，清朝有制：凡丢失国之寸土者，皆不得立碑。足以看出，中国的传统观念中，对于被侵占/吞并/收购都是不光彩的，甚至可耻的，所以也会下意识表现出一种鄙视的态度。

但其实这种想法多少有些偏激。熟读历史的人可能都知道，宋朝从建立到灭亡，燕云十六州一直都在大辽手中。而后人在评价这段历史的时候，大多认为燕云十六州的丢失是大宋灭亡的一个重要原因。但是，历史没告诉我们的是，如果建国初期，燕云十六州不在大辽手中，辽送之间也许就不会延续百年和平，可能宋朝也根本不会存世300多年。

商场如战场，在战场上，胜败乃兵家常事。所以寸土必争并非商场的标准。商业伦理发展至今，评判的标准大致是落在最大化股东权益与兼顾股东与利益相关者权益之间。

被收购是否耻辱，只要看股东和利益相关者的权益是否受侵犯就可以了。具体到这个案例，基本是个皆大欢喜的结果，没什么好羞耻的。

所以，我等屁民，站在旁观的角度，看看热闹就好。因为有些收购要考量的因素很多，比如股东的想法，CEO的KPI，通过收购公司来获得专利，甚至是通过收购公司来挖人也有可能呢。

而且，很多公司，或者公司的创始人，在被收购了之后也有很好的发展。比如FoxMail腾讯收购之后的张小龙。无论是对于腾讯，还是对于张小龙，还是对于FoxMail，这都是最好的归属。

总之呢，很多事，我们看不清，就不看，不听，不说好了。当你凝望深渊的时候，深渊也在凝望你。

被微软收购是一件坏事吗？

最近几年，微软主导了很多次大的收购案，但是据说被收购的公司的现状好像都不是很好。我们看下最近几年和微软有关的重大收购和投资案：

2016年06月13日，微软262亿收购领英。

2014年9月，微软宣布25亿美元并购瑞典游戏开发商Mojang，包含公司所有权和游戏(如该公司所开发的《我的世界》)的知识产权。

2013年9月，微软宣布72亿美元收购诺基亚设备和服务部门，包括获得诺基亚专利授权和品牌使用，2014年4月份完成收购业务。

2012年6月，微软12亿美元收购了企业内部社交网络Yammer。

2011年5月，微软斥资85亿美元收购Skype，这是微软当时最大的一笔收购案。

2007年10月，微软以150亿美元的估值，对Facebook投资2.4亿美元，换取了后者1.6%的股份。

以上的被收购及投资的公司中，除了被投资的Facebook活的还挺滋润以外，其他几家好像在被收购之后并没有什么大的突破，甚至很多都有所下滑。

这是不是说明，被微软收购的公司，都冥冥之中受到了诅咒？被微软收购，难道就是一家公司最大败笔嘛？

但其实，关于GitHub的这场收购案，笔者并不认为一定是件坏事。为什么这么说呢，我们先来看一下GitHub当前的现状。

GitHub现状

GitHub 发展至今，已经成为了目前世界上规模最大的 Hacker 社区，GitHub目前拥有2700万开发者，托管多达8000万代码，而这也成为了 GitHub 最大的财富。

虽然社区发展一帆风顺，但 GitHub 作为一家商业公司的运营，是一直收到外界质疑的。

目前，GitHub 的盈利方式主要有三种：

1.代码托管服务盈利

2.数据沉淀和云储存盈利

3.企业猎头招聘盈利

虽然已经有很好的盈利模式了，但是同样是报道GitHub被微软收购的彭博社也曾经报道过：

虽然软件开发工具提供商GitHub获得了风险投资的青睐，但该公司的开支似乎有些过度，营收增幅却不够强劲，这也给其未来的前景蒙上了一层阴影。

当然，这是一份2016年的报道，如今已经是2018年了，如今GitHub的现状是否有所好转呢？情况也许不容乐观。

因为，在2017年8月份，CEO 万斯特拉斯宣布辞任。而在那之后，GitHub 一直没有找到合适的 CEO 继任者。

一方面群龙无首，一方面财政赤字。看来，GitHub也并不如表面那么风光。

为什么被微软收购并不一定是件坏事

在GitHub被微软收购的新闻出来之后，大家基本都是唱衰的，大部分人都认为GitHub要凉。

甚至有人调侃：

以上调侃显然是不符合事实的，因为GitHub在这之前就有所谓的企业版和家庭版了。

微软对于 GitHub 的态度，其实也是一个由敌对到拥抱的转变过程。纳德拉上任以后，曾经那个封闭的微软正在变得越来越开放。

我认为这并不一定是坏事的原因有三个。

一、微软对开源的态度

2014 年，微软在 GitHub 上开设了账户，而微软目前在 GitHut 上贡献了超过 1800 个存储库，连续三年保持第一。

我到GitHub上查了一下微软的开源情况，截了几张图给大家看一下：

开源项目1835个，远超过Google的1288+233 （Google + Google Samples）个和Apache的1560个。

这其实可以看出，微软在开源方面还是下了挺大的功夫的。至少在对开源社区的贡献上，微软是走在其他公司前面的。

二、微软有可能帮助GitHub实现盈利

对于一个GitHub这样的网站来说，他们在盈利方面可能并没有太多这方面的基因。

而微软在盈利方面有丰富的经验。哈哈。所以，微软可能会帮助GitHub实现盈利的。

而且，对于GitHub来说，有了微软的这笔收购资金，还有微软这个干爹的不断输血，GitHub也不用害怕资金问题导致断粮了。

三、微软有可能帮助GitHub提升技术能力

除了微软对待开源的态度、微软对Git的资金支持外。微软的技术可能也是GitHub急切需要的。

从Git上线至今的十年里，他经受过无数次攻击。甚至一直在刷新着DDOS攻击规模的记录。

美国东部时间2018年2月28日下午12点15分左右，GitHub 遭遇了可能是迄今为止最大的 DDoS 攻击，最高访问量为 1.35Tbps。

当地时间2月28日起，GitHub 经历了两次间歇性不可访问。攻击发生 10 分钟后，GitHub 向 CDN 服务商 Akamai 请求协助，访问 GitHub 的流量交由后者接管。攻击在 Akamai 介入 8 分钟后结束，GitHub 确认该网站上用户数据的机密性或完整性未受到威胁。

虽然，GitHub的工程师肯定很牛，但是微软作为全球顶尖科技公司，技术能力在各方面肯定有GitHub可以学习和借鉴的地方。

希望GitHub在变成微软的子公司之后，可以在技术能力上输出一些到GitHub，然后在网站攻防方面有所提升。

祝福与期望

总而言之，微软需要 GitHub 的社区氛围和技术资源，而 GitHub 也需要更好的经理人帮其打点，各取所需。

虽然都是为了赚钱的商业公司，但 GitHub 在程序员们心目中的形象还是比较中立、甚至是有信仰的。

希望“嫁入豪门”以后，GitHub 能不忘初心，码农们能继续在这里开心交友，哦不是，是学习和交流。

那么问题来了，GitHub被收购了，下一个即将被收购的会是谁？StackOverflow？Docker？…
